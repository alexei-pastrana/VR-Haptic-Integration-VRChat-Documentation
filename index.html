Sistema Háptico
integrado con VRChat

Guía completa para integrar dispositivos hápticos con VRChat usando UdonSharp y OSC. Funciona desde cero, sin conocimientos previos.

9Pasos
5Fases
4–8Semanas
$329+Desde USD

Esta es la versión limpia, condensada y replicable de todo el proyecto. Está escrita para que cualquier persona pueda seguirla desde cero y llegar a tener un sistema háptico funcional integrado con VRChat.

Antes de empezar

¿Qué necesitas?

Al terminar tendrás

  • Cuenta de VRChat nivel User con acceso completo
  • Entorno de desarrollo funcional — Unity + SDK + UdonSharp
  • Avatar con sistema de contactos hápticos configurado
  • Mundo básico publicado en VRChat con lógica programada
  • Dispositivo háptico (chaleco y/o guantes) integrado y funcionando
  • Capacidad de programar eventos hápticos personalizados

Fase 1

Cuenta y Plataforma

Semana 1–2

Paso 1Crear tu cuenta de VRChat

1.1 Crear la cuenta

  1. Ir a vrchat.com/home y click en "Sign Up"
  2. Completar el formulario: username, email y contraseña
  3. Verificar el correo (llega un link de VRChat)
⚠ Muy importanteCrea la cuenta en vrchat.com — NO a través de Steam ni Meta. Las cuentas de plataforma no permiten subir avatares ni mundos.

1.2 Instalar VRChat

  1. Abrir Steam y buscar "VRChat"
  2. Click en "Instalar" — es gratuito (~2GB)
  3. Al abrir, seleccionar "Sign in with VRChat account" (NO Steam ni Meta)

Paso 2Subir de nivel: Visitor → New User → User

Fase 1A — Visitor (Días 1–7)

Rutina diaria
DÍA 1-2
├── Explorar 5-10 mundos públicos distintos por sesión
├── Priorizar mundos sociales con gente activa
│   Recomendados: "The Great Pug", "Black Cat", "Midnight Rooftop"
├── Hablar con otros usuarios (micrófono o chat de texto)
├── Agregar 3-5 amigos de forma orgánica por día
└── Evitar AFK — el tiempo inactivo no cuenta

DÍA 3-7
├── Continuar explorando mundos variados (no repetir el mismo)
├── Sesiones de 1-2 horas de uso activo
└── Preparar el pago de VRC+ para el final de la semana

Fase 1B — Activar VRC+ (~Día 7)

  1. Ir a Steam → buscar "VRChat Plus"
  2. Comprar el plan mensual: $9.99 USD
  3. El boost se aplica automáticamente y acelera el Trust Rank
✓ Indicador de éxitoTu nameplate cambia de naranja (New User) a verde (User) — acceso completo para publicar contenido.
Fase 2

Entorno de Desarrollo

Semana 2–3

Paso 3Instalar Unity y el SDK de VRChat

3.1 VRChat Creator Companion (VCC)

  1. Ir a vrchat.com/home/download y descargar el VRChat Creator Companion
  2. Ejecutar el instalador — instalará automáticamente Unity Hub, Unity 2022.3.22f1 y el SDK
🚫 No actualices UnityLa versión 2022.3.22f1 es fija para VRChat. Actualizarla romperá la compatibilidad.

3.2 Crear proyecto de mundo

  1. Abrir VCC → "Create New Project" → "World Project"
  2. Nombrar el proyecto (ej: MiPrimerMundo) y click en "Open Project"

3.3 Autenticarse en el SDK

  1. En Unity: VRChat SDK → Show Control Panel → Authentication
  2. Iniciar sesión con tu cuenta nativa de VRChat

3.4 Configurar Visual Studio 2022

  1. Descargar Visual Studio 2022 Community (gratuito) — módulo: "Game development with Unity"
  2. En Unity: Edit → Preferences → External Tools → cambiar a Visual Studio 2022
  3. Click en "Regenerate project files"

Paso 4Tu primer script con UdonSharp

4.1 Crear y escribir el script

  1. En Project panel: click derecho → Create → U# Script → nombrar PruebaInicial
  2. Reemplazar el contenido con el código de abajo
C# / UdonSharp
using UdonSharp;
using UnityEngine;
using VRC.SDKBase;

public class PruebaInicial : UdonSharpBehaviour
{
    private void Start()
    {
        Debug.Log("¡UdonSharp funciona correctamente!");
    }

    public override void Interact()
    {
        Networking.LocalPlayer.TeleportTo(
            transform.position + Vector3.up * 2f,
            transform.rotation,
            VRC_SceneDescriptor.SpawnOrientation.Default,
            false
        );
    }
}

4.2 Probar en local

  1. Crear un cubo: GameObject → 3D Object → Cube
  2. Agregar componente Udon Behaviour → asignar PruebaInicial.asset
  3. Presionar Play → verificar en consola el mensaje de éxito
✓ Indicador de éxito"¡UdonSharp funciona correctamente!" en la consola sin errores rojos.
Fase 3

Contenido

Semana 3–5

Paso 5Crear y publicar tu primer mundo

5.1 Construir la escena base

  1. Suelo: GameObject → 3D Object → Plane → Scale X=5, Y=1, Z=5 → marcar como Static
  2. Iluminación: verificar que existe una Directional Light (Mode: Mixed)
  3. Scene Descriptor: VRChat SDK → Builder → "Add a VRCSceneDescriptor"
  4. Spawn Point: crear GameObject vacío SpawnPoint a Y=1 → arrastrarlo al campo Spawns
  5. Respawn Height: configurar a -10 en el Scene Descriptor

5.2 Hornear la iluminación

Unity
Window → Rendering → Lighting
→ Activar: Baked Global Illumination
→ Desactivar: Auto Generate
→ Click en "Generate Lighting" y esperar

5.3 Subir el mundo

  1. SDK → Builder → verificar sin errores rojos
  2. Primero "Build & Test" para prueba local
  3. Luego "Build & Upload" → nombre, descripción, Capacity: 8, Visibility: Private
✓ Indicador de éxitoEl mundo aparece en tu lista de mundos dentro de VRChat.

Paso 6Avatar con sistema de contactos

6.1 Avatar base de bHaptics

  1. Ir a github.com/bhaptics/VRChatOSC y descargar el .unitypackage del avatar de ejemplo
  2. En Unity: Assets → Import Package → Custom Package → importar todo
ℹ NotaEl avatar de bHaptics ya tiene todos los Contact Receivers configurados con los nombres de parámetros correctos. Es el punto de partida más eficiente.

6.2 Verificar Contact Receivers

  1. Seleccionar avatar en Hierarchy → expandir hasta los huesos del torso
  2. Verificar en cada hueso: Local Only ✅, Receiver Type: Constant, Parameter: nombre del motor

6.3 Subir el avatar

  1. SDK → Builder → sin errores rojos → nombre + thumbnail
  2. Visibility: Private → "Build & Upload"
Fase 4

Integración Háptica

Semana 5–7

Paso 7Conectar el dispositivo háptico

Para chaleco bHaptics

  1. Descargar bHaptics Player desde bhaptics.com/software/pcplayer
  2. Conectar el chaleco vía Bluetooth → debe aparecer en verde
  3. Descargar bHapticsOSC desde github.com/bhaptics/VRChatOSC/releases
  4. Ejecutar bHapticsOSC.exe → verificar "Waiting for OSC data..."

Para guantes LucidGloves

  1. Instalar OpenGloves Driver desde Steam (App ID: 1574050)
  2. Conectar los guantes → abrir SteamVR → verificar que OpenGloves aparece en drivers
  3. Seleccionar puerto COM o activar Bluetooth en la UI de OpenGloves
  4. Mover dedos y verificar que la preview del driver responde

8.2 Script ZonaHaptica

C# / UdonSharp — ZonaHaptica.cs
using UdonSharp;
using UnityEngine;
using VRC.SDKBase;

public class ZonaHaptica : UdonSharpBehaviour
{
    [Header("Configuración Háptica")]
    [SerializeField] private string parametroHaptico = "bHaptics_Vest_Front_2_1";
    [SerializeField] private float intensidad = 0.8f;
    [SerializeField] private float duracion   = 0.5f;

    public override void OnPlayerTriggerEnter(VRCPlayerApi jugador)
    {
        if (!jugador.isLocal) return;
        Networking.LocalPlayer.SetAvatarParameter(parametroHaptico, intensidad);
        SendCustomEventDelayedSeconds(nameof(DesactivarHaptico), duracion);
    }

    public void DesactivarHaptico()
    {
        Networking.LocalPlayer.SetAvatarParameter(parametroHaptico, 0f);
    }
}

8.3 Agregar el trigger al mundo

  1. Crear GameObject vacío ZonaHaptica_01
  2. Agregar Box Collider → activar "Is Trigger" → escalar al tamaño deseado
  3. Agregar UdonBehaviour → asignar ZonaHaptica.asset
  4. Configurar: nombre exacto del parámetro, Intensidad: 0.8, Duración: 0.5
✓ Indicador de éxitoEl chaleco vibra exactamente al entrar a la zona, por 0.5 segundos.
Fase 5

Publicación y Pruebas

Semana 7–8

Paso 9Publicar y probar en multijugador

9.1 Publicar el mundo como público

  1. SDK → Builder → cambiar Visibility de Private a Public
  2. Agregar tags: "haptics", "technology", "demo"
  3. Click en "Build & Publish" → entrará a Community Labs

9.2 Protocolo de prueba

Pruebas
PRUEBA 1 — Contacto entre avatares
├── Usuario A toca el pecho de Usuario B
└── Resultado: chaleco de B vibra, A no

PRUEBA 2 — Zona trigger
├── Usuario A entra a la zona trigger
└── Resultado: chaleco de A vibra (Local Only, B no vibra)

PRUEBA 3 — Latencia
└── Meta: <50ms entre contacto visual y feedback físico

PRUEBA 4 — Estabilidad
├── 3-5 usuarios simultáneos
└── Verificar sin degradación del sistema
